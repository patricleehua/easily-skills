# Skill: analysis-projects

## Role

作为专业的项目分析助手，本 skill 负责对用户指定的代码项目进行全面、深入的分析，并生成结构化的分析报告。通过自动化的代码扫描、架构解析和业务逻辑理解，帮助开发者快速掌握项目全貌，降低项目理解成本，提升团队协作效率。

---

## 功能概述

本 skill 将在目标项目根目录下创建 `analysis-projects` 目录，并生成以下分析文档：

1. **快速开始.md** - 项目快速启动指南
2. **项目整体架构设计业务流程.md** - 架构设计与业务流程分析
3. **如何复刻该项目.md** - 项目复刻完整指南
4. **项目重点与市场上主流分析比对.md** - 竞品分析与技术比对

---

## 使用方式

### 基本调用

```bash
/analysis-projects [项目路径]
```

### 参数说明

- `[项目路径]`: 可选参数，指定要分析的项目目录
  - 如果不提供，则分析当前工作目录
  - 支持相对路径和绝对路径

### 使用示例

```bash
# 分析当前项目
/analysis-projects

# 分析指定路径的项目
/analysis-projects /path/to/your/project

# 分析相对路径的项目
/analysis-projects ./my-app
```

## 工作流程

### 第一步：项目扫描与识别

1. 扫描项目目录结构
2. 识别项目类型（前端/后端/全栈/移动端等）
3. 检测技术栈（框架、语言、依赖库等）
4. 分析项目规模和复杂度

### 第二步：深度代码分析

1. 解析入口文件和配置文件
2. 分析模块依赖关系
3. 提取核心业务逻辑
4. 识别设计模式和架构模式
5. 分析数据流和状态管理

### 第三步：生成分析文档

在项目根目录下创建 `analysis-projects` 目录，并生成四份核心文档。

### 第四步：质量检查

确保所有文档符合以下标准：

- 使用标准 Markdown 格式
- 不包含 emoji 表情符号
- 内容结构清晰、逻辑严谨
- 信息准确、可操作性强

## 分析能力

### 支持的项目类型

- **前端项目**: React, Vue, Angular, Next.js, Nuxt, Svelte 等
- **后端项目**: Node.js, Python (Django/Flask), Java (Spring), Go, Ruby on Rails 等
- **移动端**: React Native, Flutter, Swift, Kotlin 等
- **全栈项目**: MEAN, MERN, JAMStack 等
- **工具库**: npm packages, Python packages 等

### 分析维度

1. **代码质量**: 代码规范、复杂度、可维护性
2. **架构设计**: 分层结构、模块化、解耦程度
3. **性能**: 潜在性能瓶颈、优化空间
4. **安全性**: 安全漏洞、最佳实践遵循
5. **可扩展性**: 未来扩展能力评估
6. **文档完整性**: 现有文档质量评估

## 分析策略

### 针对大型项目

根据用户的 CLAUDE.md 指示：

- 采用针对性分析，避免全文检索
- 使用切面定位方式获取关键代码片段
- 优先分析：
  - 配置文件（package.json, requirements.txt 等）
  - 入口文件（main.js, index.js, app.py 等）
  - 路由定义文件
  - 核心业务模块
  - README 和现有文档

### 分析优先级

1. **高优先级**: 项目配置、入口文件、核心业务逻辑
2. **中优先级**: 工具函数、辅助模块、测试文件
3. **低优先级**: 静态资源、第三方库、构建产物

---

## 注意事项

1. **避免过度分析**: 对于大型项目，聚焦核心功能和关键模块
2. **保护敏感信息**: 不在文档中暴露密钥、密码等敏感数据
3. **尊重项目结构**: 不修改原项目代码，仅在根目录下创建分析目录
4. **保持客观**: 技术评价基于事实，避免主观偏见
5. **版本兼容**: 记录分析时的依赖版本，避免未来版本差异问题

- 在 15 分钟内理解项目核心功能

## 技术实现要点

### 使用的工具

- **Glob**: 查找特定模式的文件（配置文件、入口文件等）
- **Grep**: 搜索关键代码模式和业务逻辑
- **Read**: 读取关键文件内容
- **Write**: 生成分析文档
- **Bash**: 执行必要的命令（如获取依赖树）

### 分析算法

1. **依赖图分析**: 构建模块依赖关系图
2. **关键路径识别**: 找出核心业务流程路径
3. **复杂度评估**: 计算代码复杂度指标
4. **模式识别**: 识别常见设计模式和反模式
