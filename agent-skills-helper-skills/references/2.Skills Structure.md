## 1. Skills 的目录结构与命名规范

在支持 Agent Skills 的工具中，Skills 通常以“文件夹集合”的形式存在。每个 Skill 是一个独立文件夹，至少包含一个定义文件 `skill.md`。

典型目录（以 Claude Code 为例）：

```text
<project_root>/
  .claude/
    skills/
      <skill_name>/
        SKILL.md
        (resources...)
```

每个 Skill 文件夹至少需要：

1. Skill 文件夹存在`skill_name`
2. Skill 定义文件存在且命名为 `SKILL.md`
3. SKILL.md 内包含可被识别的元数据块（以及可选的指令内容）



## 2. 全局 Skills 与项目 Skills

### 6.1 项目级 Skills

放在项目目录下，仅对当前项目生效：

```text
<project_root>/.claude/skills/...
```

### 2.2 全局 Skills

放在工具的用户配置目录下，可对所有项目生效（具体路径因工具不同而异）。其本质是将 skills 文件夹放进工具的全局配置位置。

迁移技能包到其他工具（例如 Codex）的方式通常是将技能目录放到该工具识别的 skills 路径中，或将 `.claude` 改为该工具的配置目录名（例如 `.codex`）。

## 3. 资源层的推荐组织方式

开放标准中推荐的资源目录结构：

```text
<skill_name>/
  SKILL.md
  scripts/
  references/
  assets/
```

- scripts：可执行脚本（例如 Python、Node、Shell）
- references：参考文档、范文、规范、示例等
- assets：图片、图标、模板文件、静态资源
