## 1.通用 Skill 编写规范（推荐）

### 1.1 元数据的写法要点

元数据中最关键是“调用时机”，应写清楚触发条件。例如：

- 当用户提供某类文件（SRT、会议纪要、日志）时调用
- 当用户表达某种意图（不会写、没选题、要生成方案）时调用
- 当用户要求特定输出格式时调用

### 1.2 指令层的写法要点

建议包含以下部分：

- 角色设定（你是谁）
- 任务定义（要做什么）
- 输入假设（用户会提供什么）
- 输出结构（必须的标题、表格、字段）
- 约束与边界（禁止编造、禁止删减、必须保留原文等）
- 后续处理（若需要脚本或 MCP 工具，写清楚调用条件与步骤）

### 1.3 输出格式要稳定可复用

最好规定固定的标题、表格字段、文件命名方式，便于自动化处理与资料沉淀。

不允许输出`emoji`表情





## 2. 附录：标准 Skill 文件骨架

```
<skill_name>/
  skill.md
  license.txt
  scripts/
    run.py
  references/
    style.md
    rules.md
  assets/
    logo.png
```

skill.md 的推荐结构：

```markdown
---
name: <技能名称>
description: <调用时机与适用场景>
license: Proprietary. LICENSE.txt has complete terms
---

## 角色
...

## 输入
...

## 任务
...

## 输出格式
...

## 约束
...

## 后续处理（可选）
- scripts 执行说明
- MCP 工具调用说明
```
